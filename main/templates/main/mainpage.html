🦁 Template 🦁

기본적으로 HTML을 통해 화면을 구성한다.
프로젝트의 "앱/templates/앱/"의 경로에 html 파일을 작성한다.

Template 언어 활용
  • URL 연결

    // urls.py에서 지정한 URL의 별명을 입력
    <a href="{% url '별명' %}" >

  • 변수 사용
    1. View에 context 속성 추가
    2. context에 딕셔너리 형태로 변수 및 값 삽입
    3. render 메서드에 context 인자 넘기기
    4. html에서 {{ 변수 }}의 형식으로 사용. 변수가 리스트 혹은 딕셔너리일 경우 '.인덱스' or '.key'와 같이 사용
  • 반복문

    {% for 아이템 in 리스트 %}
       {{ 아이템 }}
    {% endfor %}

  • 조건문

    {% if 조건 %}
       참인 경우
    {% else %}
       거짓인 경우
    {% endif %}

  • 필터

    변수와 함께 {{ 변수|필터 }}의 형식으로 쓰임

Template 상속 - base.html

중복되는 코드를 project/templates의 파일(base.html)로 분리한다.
이는 base.html 내부 block 태그로 지정한 위치에 앱의 templates 코드를 삽입하는 방식으로 작동한다.
구현 방법은 아래와 같다.

  1. project/templates 경로를 추가. project/settings.py에 아래와 같이 경로 추가

    'DIRS': [os.path.join(BASE_DIR, 'project', 'templates')]

  2. project 디렉토리에 templates 폴더 생성
  3. templates 폴더에 base.html 파일 생성
  4. project/templates/base.html에 화면별 코드를 삽입할 위치에 아래 태그를 추가

    {% block 이름 %}
    {% endblock %}

  5. app/templates/app/~.html에서 base.html에 있는 부분은 모두 삭제
  6. app/templates/app/~.html을 아래와 같은 구조로 수정

    // 맨 첫 줄
    {% extends '아래 블록이 들어갈 공통 html' %}

    {% block 이름 %}
      // 공통 html에 들어갈 블록 내용
    {% block 이름 %}

Template 상속 - navbar 상속

base.html외에도 중복되는 컴포넌트가 있을 때 분리해서 관리할 수 있다. 이러한 경우 include 태그를 사용한다.

  2. project/templates/shared 폴더 생성
  3. 중복되는 부분을 project/templates/shared/~.html에 옮기기
  4. 잘라낸 곳에 {% include 'shared/컴포넌트.html' %}를 사용하여 중복되는 컴포넌트 상속시키기
정적 파일 분리
  1. project/static/css, project/static/images 폴더 생성
  2. project/static/css에는 css 파일을, project/static/images에는 이미지 파일 넣기
  3. 삽입을 원하는 파일에 아래와 같이 작성

    <!DOCTYPE html >
    {% load static %}
    ...
    <head>
      <link rel=stylesheet type="text/css" href="{% 'css/project.css' %}"

  4. 정적 파일 경로 설정

    // 오타 주의!!

    STATICFILES_DIRS = [os.path.join(BASE_DIR, 'project', 'static')]

    STATIC_ROOT = [os.path.join(BASE_DIR, 'static')]

🦁 View 🦁

View는 정해진 로직을 처리하고, Template를 리턴하는 역할을 한다.
때문에, (1) View에서 어떤 로직을 수행할 지 작성한 뒤, (2) View가 어떤 Template를 리턴할 지 지정하고 (3) View와 연결될 URL을 설정해 주어야 한다.

  1. View 로직 작성 및 리턴 template 지정

    def view이름(request):
      로직 작성
      return render(request, '리턴할 template의 경로')

  2. URL 설정

    urls.py의 urlpatterns에 url을 작성한다.

    urlpatterns = [
      path('admin/', admin.site.urls)
      path('url', 연결될 view, '별명')
    ]
